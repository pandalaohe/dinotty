## Two load-bearing discoveries (P7)
- **D-1 `set_debounce_ms` is dead code** — `src/notification.rs:49-51` defined, never called anywhere; live bell debounce is hardcoded `300`ms from `NotificationBroadcast::new()` (`:40`). Server `bell.debounce_ms` setting (`settings/mod.rs:248-252`) has ZERO backend effect today. → E2 coalesce window must be **frontend-owned** (not "~= bell debounce_ms" which is dead); wiring the dead setting is a separate optional bug-fix. Resolves codex's "biggest uncertainty" (debounce_ms semantics = neither, it's inert).
- **D-2 `/ws/notify` Lagged silent-death** (HIGH, both routes) — bounded `broadcast::channel(256)` (`notification.rs:36`) consumed by `while let Ok(event) = rx.recv().await` (`ws/mod.rs:868`) EXITS the forward task on first `RecvError::Lagged`; outer loop (`:877-885`) reacts only to client Ping/Close, so socket stays "connected" but permanently silent, no close frame → no frontend reconnect (`useNotification.ts:335-339`). `/ws/sync` avoids this via per-client `mpsc::unbounded` (`session/mod.rs:1020-1028`, retain-drop `:995-999`). Pre-existing bug NUX1 inherits + amplifies.

## Escalations — RESOLVED with user (2026-07-16)
- **E-plugin → BACKEND-ROUTED** (user: all ends must receive; read must sync). Plugin `ui.notify` (`App.vue:1253-1265`, currently frontend-only pushNotification) routes through the backend raise path → enters AttentionLedger → cross-end synced. DESIGN DETAIL: plugin notifications may be pane-less; the ledger needs a notification-level identity (synthetic `notifId`) for cross-end read-sync in addition to per-pane keys. R-e reinforcement changes from "local-only bucket" to "backend-routed + notif-id read-state".
- **E-focused-raise → ADOPT active-read model** (user: not-active → notify; activated (mouse-click into it OR typing, AND app window foreground) → read). Confirmed doable (focus + first-keystroke hooks reach App.vue). "active" = pane is current focused pane AND window has foreground/visibility. This also implements E4 (no beep while actively watching). Composes with D2 (whole-tab popup suppression toggle).
- **E-reveal → KEEP GLOBAL** (user: global jump is the app's own positioning, don't add drama). revealPane reuses existing global activate/broadcast plumbing UNCHANGED; F fix is ONLY the bug — add the missing workspace-switch + tab-bar highlight/scroll-into-view so the target tab is correctly revealed. R-h reinforcement DROPPED (no surface-scoping); keep global broadcast.
- **E-debounce → FIX IT** (user). Wire `set_debounce_ms` (`notification.rs:49-51`, currently never called) so the existing `bell.debounce_ms` setting actually takes effect; E2 coalesce window is based on this now-live value (or a per-surface presentation window). Dead-setting bug fixed as part of NUX1.

### Extra user confirmation (2026-07-16)
- **Manual tab-click marks read + cross-syncs**: clicking any tab in the tab bar (`activateTab` → B's `clearForPaneIds`, whole tab incl all split leaves) must mark that tab read AND emit backend `mark_read` (reason `tab_activate`) so other surfaces sync. All four read-triggers — manual tab click (B), notification goto (F), focus (A), first keystroke (A) — emit backend mark_read for cross-end sync.
