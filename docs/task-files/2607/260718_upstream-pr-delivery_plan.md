## 4. Stack-file mechanics (self-shrinking cross-fork PRs)

A cross-fork PR's diff is `dev...head` computed live. Build a dependent bundle branch as `upstream/dev` + cherry-pick(prerequisite commits, in original order) + cherry-pick(this bundle's commits). File PR base=`dev`. Today the diff includes prerequisites; when the maintainer merges the prerequisite PRs into `dev`, GitHub recomputes `dev...head`, drops the now-in-base commits (patch-id match), and the PR auto-shrinks to just this bundle's own delta. Each bundle PR body MUST list its prerequisite PR numbers and "merge those first; this shrinks automatically."

Build order (cherry-pick the CONTIGUOUS original commit range where possible to minimise conflicts — these were sequential commits on `feat/notification-ux`): P6 → P1 → P2 → P3a → P3b → P4a → P4b → (F tail) → P5 → (G tail); SVT1 from `feat/supervise-tabs-r2` minus CSB commits.
